Корпоративная система управления продажами на Wildberries, объединяющая Flutter-клиент и Flask-сервер с PostgreSQL. Предназначена для автоматизации ключевых процессов селлера: управление остатками (FBO/FBS), ценообразование, скидки, акции, поставки, аналитика и отчёты.

Основные возможности
1. Управление остатками и автообновление
Единая таблица товаров с отображением остатков FBO (со складов WB) и FBS (собственных складов).

Автоматическое пополнение FBS по расписанию с гибкими правилами:

Общие настройки (порог FBO, минимальный остаток FBS).

Индивидуальные настройки для каждого склада.

Настройки для тегов (например, «always_zero», «always_n», «ignore»).

Ручная корректировка остатков FBS для любого товара.

2. Цены и скидки
Просмотр и редактирование цены, скидки, скидки WB Клуба.

Отправка изменений пакетно или по одному товару.

Отображение динамики цен (гистограммы за 21 день).

Автоматический подхват текущих цен для покупателя через публичное API карточек.

3. Акции и кампании
Интеграция с API акций Wildberries (календарь промо).

Отображение рекомендуемой скидки по акции, визуальная индикация выполнения условий.

Поддержка регулярных и автоакций (через приватное API).

Управление рекламными кампаниями: старт, пауза, завершение, просмотр статистики (клики, показы, CTR, затраты).

4. Поставки и логистика
Расчёт потребности в поставке: (период поставки - дней остатка) × заказов в день.

Автоматический подбор оптимального количества коробов (задача о рюкзаке).

Экспорт потребности в Excel с возможностью ручной корректировки.

Проверка доступности коробов на складе через API.

5. Отчёты и аналитика
Ежедневные финансовые отчёты (аналог отчёта о продажах WB) с полной детализацией.

Гибкая таблица с настраиваемыми колонками, фильтрами, сортировкой, группировкой.

Построение линейных графиков по выбранным показателям (динамика продаж, комиссий и т.д.).

Экспорт в Excel, CSV, Parquet.

6. Справочники и настройки
Загрузка комиссий (кВВ) по предметам.

Тарифы коробов (логистика) по складам.

Себестоимость и дополнительные расходы (загрузка через XLSX-шаблон).

Справочник предметов (subject) и их родительских категорий.

Маппинг названий складов между разными API.

7. Логи и мониторинг
Централизованное логирование всех операций в PostgreSQL.

Long polling для реального времени (просмотр логов в браузере).

Счётчик ошибок за последние 24 часа в интерфейсе.

8. Клиент-серверная архитектура
Сервер: Flask, SQLAlchemy, APScheduler, интеграция с API Wildberries.

Клиент: Flutter (Desktop Windows), плагин PlutoGrid для таблиц, webview для встроенного браузера.

Автоматическое обновление клиента через API.

Технологический стек
Backend: Python 3.9+, Flask, PostgreSQL, SQLAlchemy, Pandas, APScheduler, Requests.

Frontend: Flutter 3.x, Dart, PlutoGrid, WebView Windows, SharedPreferences.

API Wildberries: content-api, statistics-api, marketplace-api, dp-calendar-api, seller-analytics-api, supplies-api.

Форматы: JSON, CSV, Excel (openpyxl, xlsxwriter), Parquet.
